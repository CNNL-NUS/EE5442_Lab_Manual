Running mumax3 on the HPC

SSH Connection to the HPC

  1. Setup the SSH connection to the cluster, as done previously in Lab 1 (Week 2).
  1. Login with your username and password.
  1. Access the files on the SSH Browser (FTP)
  1. The .ovf files, tables and log can be downloaded and analyzed on local machine. Use the imem0XX folder assigned to you from before.
  1. Create .mx3 files to run scripts on mumax3.

Load mumax3 interactive shell

  1. qsub : batch client to submit scripts to server Using qsub to get allotted with a server of following settings.
    qsub -I -X -l cudamem=300mb

  1. Enter the interactive shell.
    /apps/containers/scripts/umag/loadUmagTools.sh --shell -nv

  1. Once inside the container manager (Singularity), enter the profile.
    . /etc/profile

  1. Load the mumax3 module in the containe.
    module load local mumax

  1. Go to the directory with .mx3 file to run / add the relative path, and run with mumax3.
    mumax3 test.mx3

To have web interface in Firefox browser, gives a local port. Open Firefox and load the address (127.0.0.1:port). For VPN (server_ip:port).

Web Browser interface is not being used in this lab.

mumax3 –i test.mx3

Submitting mumax3 jobs

  1. In longer simulations the host will logout the user after sometime. If running in interactive mode data is lost. Submitting the jobs via qsub and shell scripts, gives a Job-ID and ensure the job runs in the background.
  1. The shell scripts run_batch.sh and run_single.sh are provided. Place them in the running directory. The shell scripts can be downloaded here.

  1. Edit the Project directory to your project path.
  1. Edit the file name in run_single. Then edit node (LandauerXX) in the run_batch script and run.
sh run_batch.sh

  1. A Job-ID will be assigned. Check status.
qstat

  1. Delete the job.
 qdel "job-id"
