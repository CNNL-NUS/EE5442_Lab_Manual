Running mumax3 on the HPC

Setup

Setup the SSH connection to the cluster, as done previously with the respective host IP.

172.18.72.20 Feynman Cluster to access the other nodes. Login with your username and password.

Access the files on the SSH Browser (FTP)

The .ovf files, tables and log can be downloaded and analyzed on local machine.

Create .mx3 files to run scripts on mumax3.

Loading mumax interactive shell

qsub : batch client to submit scripts to server

Using qsub to get allotted with a server of following settings: qsub -I -X -l cudamem=300mb

Enter the interactive shell:

/apps/containers/scripts/umag/loadUmagTools.sh --shell -nv

Once inside the container manager (Singularity), enter the profile: . /etc/profile

Load the mumax3 module in the container:

module load local mumax

Go to the directory with .mx3 file to run / add the relative path, and run with mumax3: mumax3 test.mx3

To have web interface in Firefox browser, gives a local port. Open Firefox and load the address (127.0.0.1:port). For VPN (server_ip:port):

mumax3 –i test.mx3

Submitting mumax3 jobs

In longer simulations the host will logout the user after sometime. If running in interactive mode data is lost.

Submitting the jobs via qsub and shell scripts, gives a Job-ID and ensure the job runs in the background.

The shell scripts run_batch.sh and run_single.sh are provided. Place them in the running directory.

Edit the Project directory to your project path.

Edit the file name in run_single. Then edit node (LandauerXX) in the run_batch script and run as:

sh run_batch.sh

A Job-ID will be assigned. Check status with: qstat

Delete the job with: qdel <job-id>

Executing mumax scripts

⮚Method 1: Type in “mumax3 myfile.mx3” in Command Prompt.

⮚Method 2: Right click the XX.mx3 script file → Open with and select mumax3.exe open and execute it.
